Otimização¶

  • Otimizar: melhorar a eficiência, reduzir custos, ajustar parâmetros, ...
  • Envolve:
    • Definir uma função objetivo, ou um residuo, a ser minimizada(o) (ou maximizada(o));
    • Incorporar possíveis restrições a essa função objetivo;
    • Métodos para encontrar o(s) mínimo(s) (ou máximo(s)) da função objetivo.
  • Exemplos:
    • Encontrar o retângulo de maior área com um certo perímetro dado.
    • Determinar o menor caminho passando por determinados pontos uma única vez (caixeiro viajante).
    • Maximir a produção de energia (elétrica, solar, eólica, ...).
    • Minimizar perdas em vários processos.

Otimização com o python¶

A função numpy.linalg.lstsq que usamos anteriormente para regressão linear é uma implementação direta do método de mínimos quadrados linear sem restrição, que leva à resolução de um problema linear associado.

Para problemas não-lineares de mínimos quadrados e de otimização em geral, o pacote scipy.optimize (Optimization and Root Finding) possui várias outras funções:

  • minimize_scalar: minimização de funções escalares de uma variável.
  • minimize: minimização (local) com ou sem restrições de funções objetivo escalares de uma ou mais variáveis.
  • least_squares: mínimos quadrados não-linear, com ou sem restrições, através de vários métodos.
  • leastsq: outra implementação, mais antiga, de resolução de probemas de mínimos quadrados não-linear.
  • curve_fit: ajuste de curvas baseado nas funções de resolução de mínimos quadrados mencionadas acima (qual delas depende do método escolhido).
  • nnls e lsq_linear: mínimos quadrados linear com restrições.
  • linprog: minimização de funções objetivo lineares, com restrições lineares, para programação linear.
  • basinhopping, brute, differential_evolution, shgo, dual_annealing: algoritmos variados de minimização global.

Veremos, a seguir, as funções least_squares, minimize, curve_fit e basinhopping.

Métodos implementados na função least_squares¶

A função least_squares utiliza um dos seguintes métodos de resolução, através da opção method:

  • ‘trf’ : Trust Region Reflective algorithm, particularly suitable for large sparse problems with bounds. Generally robust method.

  • ‘dogbox’ : dogleg algorithm with rectangular trust regions, typical use case is small problems with bounds. Not recommended for problems with rank-deficient Jacobian.

  • ‘lm’ : Levenberg-Marquardt algorithm as implemented in MINPACK. Doesn’t handle bounds and sparse Jacobians. Usually the most efficient method for small unconstrained problems.

Neste caso em particular, poderíamos usar, ainda, o minimize_scalar, já que temos um problema unidimensional.

Ajuste de curva via mínimos quadrados não-linear¶

A função objetivo de mínimos quadrados não linear é uma versão não linear dos mínimos quadrados.

Mais precisamente, dada uma função de ajuste $f_\beta$, com $\beta=(\beta_1, \ldots, \beta_k)$ representando os parâmetros dessa função, e dado uma amostra $(x_i, y_i)_i$, $i=1, \ldots, N$, temos os resíduos $\Delta_i= y_i - f_\beta(x_i)$ e o objetivo é encontrar

$$ \hat\beta = \text{argmin } F(\beta) = \frac{1}{2} \sum_i \rho\left(\left(y_i - f_\beta(x_i)\right)\right)^2, \; \text{com uma restrição} \; l_\beta \leq \beta \leq u_\beta. $$

A notação "argmin" significa que buscamos o argumento do problema de minimização, ou seja, o ponto de mínimo, não o valor mínimo da função.

Opções para a função de "perda"¶

A função de perda $\rho$ visa reduzir a influência de outliers nos dados.

Há várias funções $\rho$ implementadas e é possível usar funções definidas pelo usuário:

  • "linear" (default), com $\rho(z) = z$: mínimos quadrados clássico).

  • "soft_l1", com $\rho(z) = 2 ((1 + z)^{1/2} - 1)$. Aproximação suave para a norma $L^1$, visando diminuir a influência de outliers.

  • "huber", com $\rho(z) = z$, se $z <= 1$, ou $2z^{1/2} - 1$. Similar ao "soft_l1".

  • "cauchy", com $\rho(z) = \ln(1 + z)$. Diminui severamente a influência de outliers, mas dificulta o processo de otimização.

  • "arctan" : $\rho(z) = \arctan(z)$. Similar ao "cauchy".

Visualizando as funções de perda implementadas¶

Funções de ajuste no scipy.optimize¶

Temos, de início, três opções para implementar a minimização da soma do quadrado dos resíduos $\Delta_i = y_i - f_\beta(x_i)$,

$$ \text{argmin } F(\beta) = \frac{1}{2} \sum_i \rho\left(\left(y_i - f_\beta(x_i)\right)\right)^2, \; \text{com uma restrição} \; l_\beta \leq \beta \leq u_\beta. $$

  • Função least_squares: passamos explicitamente uma escolha de $\rho$, dentre algumas já implementads, e um vetor com os resíduos $\Delta_i$, que podem ser $y_i - f_\beta(x_i)$ ou diferente.

  • Função minimize: mais flexível, mas devemos escrever explicitamente a função objetivo $F(\beta)$, que pode ser da forma acima ou diferente.

  • Função curve_fit: definimos livremente $\rho$ e passamos separadamente a função de ajuste $f_\beta$ e os dados $x_i$, $y_i$, para que a função curve_fit monte a função objetivo $F(\beta)$ e chame a função least_squares para fazer o ajuste.

Métodos de otimização das funções¶

Os métodos de otimização da função minimize foram mencionados acima.

Os métodos das funções least_squares e curve_fit são

  • "trf" : Trust Region Reflective algorithm, particularly suitable for large sparse problems with bounds. Generally robust method. Default if bounds are given.

  • "dogbox" : dogleg algorithm with rectangular trust regions, typical use case is small problems with bounds. Not recommended for problems with rank-deficient Jacobian.

  • "lm" : Levenberg-Marquardt algorithm as implemented in MINPACK. Doesn’t handle bounds and sparse Jacobians. Usually the most efficient method for small unconstrained problems. Default if bounds are not given.

def objetivo(a, x, y):

Ajustando os parâmetros do modelo SIR a dados reais¶

Vamos, agora, ajustar os parâmetros $\beta$ e $\gamma$ do modelo SIR aos dados de uma epidemia de Influenza em um escola de meninos (internato) no norte da Inglaterra, com um total de 763 residentes, em um período de duas semanas, em que, a cada dia, o número de infectados é dado pela lista

$$ \text{Infectados a partir do três} = [25, 75, 228, 297, 259, 235, 192, 126, 71, 28, 9, 7]. $$

Fonte: Brauer & Castillo-Chavez (2012).

Há uma lista um pouco diferente de infectados, que inclui desde o dia zero:

$$ \text{Infectados a partir do dia zero} = [1, 3, 7, 25, 72, 222, 282, 256, 233, 189, 123, 70, 25, 11, 4]. $$

Fonte: Ledder(2013)

Vamos usar, abaixo, os primeiros dados, de Brauer & Castillo-Chavez (2012).

Ajuste¶

Nesse caso, o modelo nos dá, a partir dos dados iniciais e dos parâmetros, uma solução $(S(t), I(t), R(t))$. Mas só temos dados dos números de infectados $I_j$, a cada dia $j=0, \ldots, 14$. Assim, a nossa função objetivo só pode ser construída em cima de $I(t)$ e dos dados $(t_j, I_j)$.

Em algumas situações, as próprias condições iniciais também podem entrar como parâmetros a serem ajustados, mas no caso mais controlado acima, vamos fixar as seguinte condições iniciais:

  • $I(0) = I_j = 1$ residente;
  • $S(0) = 763 - 1 = 762$ residentes; e
  • $R(0) = 0$ residentes.

Definimos a taxa de evolução do sistema $(S,I)$ do modelo SIR:

In [22]:
def taxa_de_evolucao_SIR(t, X, N, beta, gamma):
    S, I = X
    dXdt = [- beta*I*S/N, beta*I*S/N - gamma*I]
    return dXdt

def solve_SIR(t_eval, beta, gamma):
    """
    Retorna o número de infectados no instante t pelo SIR com os parâmetros dados.
    """
    # população total
    N = 763

    # população inicial em cada estágio
    S0 = 762 # suscetíveis
    I0 = 1 # infectados
    R0 = N - S0 - I0 # recuperados

    # populacao inicial para o sistema reduzido
    X0 = [S0, I0]
    
    # resolvendo a equação diferencial
    X = solve_ivp(taxa_de_evolucao_SIR, t_span=[0, t_eval[-1]], y0 = X0, t_eval = t_eval, args=(N, beta, gamma))
    
    # retornando o vetor de valores da segunda variável, correspondendo ao número de infectados 
    # em cada instante
    sol = SIR(X.y[0], X.y[1], N-X.y[0]- X.y[1])
    return sol
