Otimização¶

  • Otimizar: melhorar a eficincia, reduzir custos, ajustar parmetros, ...
  • Envolve:
    • Definir uma funo objetivo, ou um residuo, a ser minimizada(o) (ou maximizada(o));
    • Incorporar possveis restries a essa funo objetivo;
    • Mtodos para encontrar o(s) mnimo(s) (ou mximo(s)) da funo objetivo.
  • Exemplos:
    • Encontrar o retngulo de maior rea com um certo permetro dado.
    • Determinar o menor caminho passando por determinados pontos uma nica vez (caixeiro viajante).
    • Maximir a produo de energia (eltrica, solar, elica, ...).
    • Minimizar perdas em vrios processos.

Otimização com o python¶

A funo numpy.linalg.lstsq que usamos anteriormente para regresso linear uma implementao direta do mtodo de mnimos quadrados linear sem restrio, que leva resoluo de um problema linear associado.

Para problemas no-lineares de mnimos quadrados e de otimizao em geral, o pacote scipy.optimize (Optimization and Root Finding) possui vrias outras funes:

  • minimize_scalar: minimizao de funes escalares de uma varivel.
  • minimize: minimizao (local) com ou sem restries de funes objetivo escalares de uma ou mais variveis.
  • least_squares: mnimos quadrados no-linear, com ou sem restries, atravs de vrios mtodos.
  • leastsq: outra implementao, mais antiga, de resoluo de probemas de mnimos quadrados no-linear.
  • curve_fit: ajuste de curvas baseado nas funes de resoluo de mnimos quadrados mencionadas acima (qual delas depende do mtodo escolhido).
  • nnls e lsq_linear: mnimos quadrados linear com restries.
  • linprog: minimizao de funes objetivo lineares, com restries lineares, para programao linear.
  • basinhopping, brute, differential_evolution, shgo, dual_annealing: algoritmos variados de minimizao global.

Veremos, a seguir, as funes least_squares, minimize, curve_fit e basinhopping.

Métodos implementados na função least_squares¶

A funo least_squares utiliza um dos seguintes mtodos de resoluo, atravs da opo method:

  • trf : Trust Region Reflective algorithm, particularly suitable for large sparse problems with bounds. Generally robust method.

  • dogbox : dogleg algorithm with rectangular trust regions, typical use case is small problems with bounds. Not recommended for problems with rank-deficient Jacobian.

  • lm : Levenberg-Marquardt algorithm as implemented in MINPACK. Doesnt handle bounds and sparse Jacobians. Usually the most efficient method for small unconstrained problems.

Ajuste de curva via mínimos quadrados não-linear¶

A funo objetivo de mnimos quadrados no linear uma verso no linear dos mnimos quadrados.

Mais precisamente, dada uma funo de ajuste $f_\beta$, com $\beta=(\beta_1, \ldots, \beta_k)$ representando os parmetros dessa funo, e dado uma amostra $(x_i, y_i)_i$, $i=1, \ldots, N$, temos os resduos $\Delta_i= y_i - f_\beta(x_i)$ e o objetivo encontrar

$$ \hat\beta = \text{argmin } F(\beta) = \frac{1}{2} \sum_i \rho\left(\left(y_i - f_\beta(x_i)\right)\right)^2, \; \text{com uma restrio} \; l_\beta \leq \beta \leq u_\beta. $$

A notao "argmin" significa que buscamos o argumento do problema de minimizao, ou seja, o ponto de mnimo, no o valor mnimo da funo.

Opções para a função de "perda"¶

A funo de perda $\rho$ visa reduzir a influncia de outliers nos dados.

H vrias funes $\rho$ implementadas e possvel usar funes definidas pelo usurio:

  • "linear" (default), com $\rho(z) = z$: mnimos quadrados clssico).

  • "soft_l1", com $\rho(z) = 2 ((1 + z)^{1/2} - 1)$. Aproximao suave para a norma $L^1$, visando diminuir a influncia de outliers.

  • "huber", com $\rho(z) = z$, se $z <= 1$, ou $2z^{1/2} - 1$. Similar ao "soft_l1".

  • "cauchy", com $\rho(z) = \ln(1 + z)$. Diminui severamente a influncia de outliers, mas dificulta o processo de otimizao.

  • "arctan" : $\rho(z) = \arctan(z)$. Similar ao "cauchy".

Funções de ajuste no scipy.optimize¶

Temos, de incio, trs opes para implementar a minimizao da soma do quadrado dos resduos $\Delta_i = y_i - f_\beta(x_i)$,

$$ \text{argmin } F(\beta) = \frac{1}{2} \sum_i \rho\left(\left(y_i - f_\beta(x_i)\right)\right)^2, \; \text{com uma restrio} \; l_\beta \leq \beta \leq u_\beta. $$

  • Funo least_squares: passamos explicitamente uma escolha de $\rho$, dentre algumas j implementads, e um vetor com os resduos $\Delta_i$, que podem ser $y_i - f_\beta(x_i)$ ou diferente.

  • Funo minimize: mais flexvel, mas devemos escrever explicitamente a funo objetivo $F(\beta)$, que pode ser da forma acima ou diferente.

  • Funo curve_fit: definimos livremente $\rho$ e passamos separadamente a funo de ajuste $f_\beta$ e os dados $x_i$, $y_i$, para que a funo curve_fit monte a funo objetivo $F(\beta)$ e chame a funo least_squares para fazer o ajuste.

Métodos de otimização das funções¶

Os mtodos de otimizao da funo minimize foram mencionados acima.

Os mtodos das funes least_squares e curve_fit so

  • "trf" : Trust Region Reflective algorithm, particularly suitable for large sparse problems with bounds. Generally robust method. Default if bounds are given.

  • "dogbox" : dogleg algorithm with rectangular trust regions, typical use case is small problems with bounds. Not recommended for problems with rank-deficient Jacobian.

  • "lm" : Levenberg-Marquardt algorithm as implemented in MINPACK. Doesnt handle bounds and sparse Jacobians. Usually the most efficient method for small unconstrained problems. Default if bounds are not given.

Ajustando os parâmetros do modelo SIR a dados reais¶

Vamos, agora, ajustar os parmetros $\beta$ e $\gamma$ do modelo SIR aos dados de uma epidemia de Influenza em um escola de meninos (internato) no norte da Inglaterra, com um total de 763 residentes, em um perodo de duas semanas, em que, a cada dia, o nmero de infectados dado pela lista

$$ \text{Infectados a partir do trs} = [25, 75, 228, 297, 259, 235, 192, 126, 71, 28, 9, 7]. $$

Fonte: Brauer & Castillo-Chavez (2012).

H uma lista um pouco diferente de infectados, que inclui desde o dia zero:

$$ \text{Infectados a partir do dia zero} = [1, 3, 7, 25, 72, 222, 282, 256, 233, 189, 123, 70, 25, 11, 4]. $$

Fonte: Ledder(2013)

Vamos usar, abaixo, os primeiros dados, de Brauer & Castillo-Chavez (2012).

Ajuste¶

Nesse caso, o modelo nos d, a partir dos dados iniciais e dos parmetros, uma soluo $(S(t), I(t), R(t))$. Mas s temos dados dos nmeros de infectados $I_j$, a cada dia $j=0, \ldots, 14$. Assim, a nossa funo objetivo s pode ser construda em cima de $I(t)$ e dos dados $(t_j, I_j)$.

Em algumas situaes, as prprias condies iniciais tambm podem entrar como parmetros a serem ajustados, mas no caso mais controlado acima, vamos fixar as seguinte condies iniciais:

  • $I(0) = I_j = 1$ residente;
  • $S(0) = 763 - 1 = 762$ residentes; e
  • $R(0) = 0$ residentes.

Definimos a taxa de evoluo do sistema $(S,I)$ do modelo SIR:

In [22]:
def taxa_de_evolucao_SIR(t, X, N, beta, gamma):
    S, I = X
    dXdt = [- beta*I*S/N, beta*I*S/N - gamma*I]
    return dXdt

def solve_SIR(t_eval, beta, gamma):
    """
    Retorna o nmero de infectados no instante t pelo SIR com os parmetros dados.
    """
    # populao total
    N = 763

    # populao inicial em cada estgio
    S0 = 762 # suscetveis
    I0 = 1 # infectados
    R0 = N - S0 - I0 # recuperados

    # populacao inicial para o sistema reduzido
    X0 = [S0, I0]
    
    # resolvendo a equao diferencial
    X = solve_ivp(taxa_de_evolucao_SIR, t_span=[0, t_eval[-1]], y0 = X0, t_eval = t_eval, args=(N, beta, gamma))
    
    # retornando o vetor de valores da segunda varivel, correspondendo ao nmero de infectados 
    # em cada instante
    sol = SIR(X.y[0], X.y[1], N-X.y[0]- X.y[1])
    return sol
